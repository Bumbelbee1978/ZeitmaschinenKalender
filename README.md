Aufgabe & Zielsetzung:

Die ursprüngliche Aufgabenstellung umfasste lediglich die Konzeption und Erstellung eines statischen Mockups (visueller Entwurf). 
Über diese Anforderung hinaus habe ich das Konzept eigenständig weiterentwickelt und ein funktionales Bedienmodell entworfen. 
Ein zentrales Ziel war dabei ein konsequenter „Offline-First-Ansatz“: Die Anwendung realisiert eine vollständige Unabhängigkeit 
von externen Cloud-Lösungen wie Google Calendar oder anderen Drittanbietern, um Datensouveränität und autarke Nutzbarkeit zu gewährleisten.

Bedienkonzept & Funktionalität:

Das User Interface (UI) setzt auf ein intuitives und haptisches Steuerungssystem:
Navigation & Datumswahl: Die Einstellung des Datums erfolgt flexibel über das Mausrad sowie die linke und rechte Maustaste.
Interaktive Steuerung: Durch das Betätigen (Herunterziehen) eines Hebels öffnet sich ein integriertes Detailfenster.
Terminverwaltung: Innerhalb dieses Fensters können vorhandene Einträge eingesehen, bearbeitet oder neue Termine angelegt werden.
Intelligente Automatisierung: Das System bietet die Möglichkeit, Termine bei der Erstellung effizient zu duplizieren. Dabei werden 
verschiedene Rhythmen unterstützt: jährlich (z. B. für Geburtstage), monatlich oder wöchentlich (spezifisch für Werktage von Mo–Fr).
